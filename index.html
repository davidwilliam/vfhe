Example #1

In this example, given $p,q,r$ and $n = pqr$, we have a message space in $\mathbb{Z}_{q}$. For all $m \in \mathbb{Z}_{q}$, encryption is given by $c = m + dpq \bmod n$ for $d \leftarrow {0,\ldots,r-1}$. We set $p=5,q=3,r=4$.

Decryption is given by $m = c \bmod q$. Homomorphic addition and multiplication are given by $c_{3} = c_1 + c_2 \bmod n$ and $c_{4} = c_1 c_2 \bmod n$ for all $c_1,c_2,c_3,c_4 \in \mathbb{Z}_{n}$.

In this simple example we can identify the patterns of linear and non-linear mappings as well as encryption and homomorphic correctness using color codes.

See Example #1

Example #2

This example is similar to Example #1 except this time we allow a larger message space. We set $p=5,q=7,r=4$.

With more messages to encrypt, we see different patterns for non-linear mappings, still keeping the cyclic nature of the results.

This modification does not affect correctness.

See Example #2

Example #3

In this example, we force a situation where correctness will fail. Encrption is given by $c = ((dq + m) + dpq) \bmod n$ and decryption is given by $m = (c \bmod p) \bmod q$. We set $p=11,q=5,r=4$.

This configuration generates large tables so the page might take a while to fully load.

As expected, correctness fails for most cases. But even then we can see patterns in the failures.

See Example #3

Example #5

In this example, encryption is given by $c = d1*q + m + d2 * p \bmod n$ for $d_1 \leftarrow \{1,\ldots,r-1\}$ and $d_2 \leftarrow \{r,\ldots,2r-1\}$. Decryption is given by $m = (c \bmod p) \bmod q$.

We have now $r(r - 1)$ distinct encryptions for each one of the $q$ plaintexts.

Correctness fails in many cases. Still, it is interesting to see that correctness fails always in the same pattern, which creates identical shapes, one for addition and another one for multiplication.

See Example #4
